Teil 2: Kurzantworten
Antworte in 1–2 Sätzen.

Was ist der Unterschied zwischen einem Natural Key und einem Surrogate Key?
- Ein Natural Key hat einen Bezug auf echte Eigenschaften einer Person, eines Produktes etc. Zum Beispiel wenn wir eine User Tabelle haben und für den Primary Key die E-Mail Adresse des Users benutzen oder die IBAN eines Kunden in einer Tabelle für Kunden in einer Bank
- Ein Surrogate Key oder Ersatz Key im deutschen, kann eine beliebige Zahl oder ähnliches sein die keinen Bezug auf echte Eigenschaften/Werte haben

Warum ist es sinnvoll, in einer Tabelle einen Primary Key zu verwenden?
- Damit jeder Eintrag bzw. Zeile/Row einzigartig ist um zu versichern dass man gezielt einer dieser Eitnräge auswählen kann.

Erkläre kurz, was ein Composite Key ist und wann er verwendet wird.
- Ein Composit key besteht aus mehreren Primary Keys, da ein einzelner dieser Keys nicht versichert die Zeile/Row eindeutig identifizieren zu können


Teil 3: Zuordnungsaufgabe
Ordne zu – welche Begriffe passen zu den Beschreibungen:

Beschreibung	                                                              Begriff
Eine Kombination mehrerer Spalten, die zusammen eine Zeile eindeutig machen -- Composit Key	
Ein Schlüssel ohne Bezug zu echten Eigenschaften (z. B. eine zufällige ID) -- Surrogate Key	
Ein Attribut, das aus realen Daten stammt, z. B. E-Mail-Adresse	-- Natural Key
Ein Feld, das auf den Primary Key einer anderen Tabelle verweist -- Foreign Key
